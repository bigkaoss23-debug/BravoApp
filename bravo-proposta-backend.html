Questo documento descrive l'architettura tecnica consigliata per il sistema BRAVO — Centro de Mando: un hub di comando intelligente, multi-dispositivo, con agenti AI specializzati e integrazione completa con gli strumenti già in uso (Google Drive, Google Calendar, Canva). 02 — Analisi Comparativa
Supabase vs Google Drive
Google Drive è costruito per gli esseri umani che lavorano con file. Supabase è costruito per le applicazioni e gli agenti AI che lavorano con dati strutturati. Non si escludono — si completano.
Caratteristica Supabase Google Drive / Sheets
Tipo di sistema Database applicativo (PostgreSQL) Storage di file per umani
Aggiornamenti real-time Sì — ogni modifica visibile istantaneamente su tutti i dispositivi No — bisogna aggiornare la pagina manualmente
Supporto multi-agente Sì — ogni agente ha la sua chiave con permessi precisi Impossibile gestire permessi separati per agente
Query e ricerca dati SQL in millisecondi — "mostra tutti i task in scadenza questa settimana" Bisogna iterare riga per riga in Sheets — lento e fragile
Conflitti simultanei Zero — transazioni SQL garantiscono la consistenza Frequenti — due persone che modificano lo stesso Sheet si sovrascrivono
Velocità API ~50ms per chiamata ~500ms+ per chiamata Drive API
Sicurezza per ruolo Row Level Security — ogni utente/agente vede solo i dati autorizzati Permissioni solo a livello di file, non di singola riga
Automazioni e webhook Sì — un agente può reagire automaticamente alle azioni degli altri Non disponibile
AI e ricerca semantica Sì — pgvector integrato per ricerca intelligente su tutti i dati Non disponibile
Autenticazione utenti Auth integrata con Google, email, ruoli personalizzati Solo tramite Google Workspace — nessun controllo granulare
Costo base Gratuito fino a 50.000 righe — poi ~$25/mese Gratuito — ma non è un database
Funzioni serverless Edge Functions integrate — logica agenti sul cloud senza server Google Apps Script — limitato e complesso
03 — Strategia Dati
Cosa va dove — approccio 50/50
Le due piattaforme non si escludono. Google Drive rimane il luogo dei file. Supabase diventa il luogo dei dati strutturati. Ogni strumento fa ciò per cui è stato costruito.
SB
Supabase — Dati strutturati
  • Progetti, clienti, stati, deadlines
  • Card Kanban, colonne, assegnazioni
  • Obiettivi mensili e trimestrali per progetto
  • Registro decisioni e storico azioni
  • Messaggi chat del team
  • Configurazione e permessi degli agenti AI
  • Calendario eventi e scadenze
  • KPI, metriche, progressi campagne
GD
Google Drive — File e documenti
  • Brief clienti in PDF e Google Docs
  • Asset grafici, foto, video prodotti
  • Brand guidelines e moodboard
  • Presentazioni e proposte commerciali
  • Materiali di rodaggio e cataloghi
  • Report esportati da Supabase
  • Template newsletter e creatività
  • Contratti e documenti legali

04 — Multi-Agente
Come comunicano gli agenti tra loro
Ogni agente AI ha la propria identità e permessi. Tutti leggono e scrivono su Supabase. Quando un agente agisce, gli altri vengono notificati automaticamente. Il CEO vede tutto in BRAVO senza dover fare nulla.
01 Agente Calendario → rileva nuova scadenza per Rossi Srl
└→ salva evento in Supabase + sincronizza Google Calendar
02 Supabase → notifica in real-time tutti i client connessi
└→ BRAVO mostra il nuovo evento al CEO istantaneamente
03 Agente Personale → riceve notifica, verifica disponibilità team
└→ suggerisce assegnazione a Carlos Lage (carico libero)
04 Agente Marketing → associa la scadenza alla campagna Q2 attiva
└→ card Kanban aggiornata automaticamente nel tablero
05 CEO in BRAVO → approva con un click, tutto il sistema si aggiorna
05 — Stack Tecnologico
Le tecnologie del sistema
Lo stack è scelto per essere solido, economico da gestire e senza dipendenza da fornitori costosi. Tutte le tecnologie hanno piani gratuiti adeguati per la fase di avvio.
Frontend
BRAVO HTML/JS
Interfaccia attuale, da evolvere. Nessun framework necessario nella fase iniziale.
Database + Real-time
Supabase
PostgreSQL hosted + API REST + Real-time + Auth + Storage. Gratuito fino a ~50k righe.
Storage file
Supabase + Google Drive
Drive per lavoro quotidiano del team. Link salvati in Supabase per accesso dall'app.
Backend agenti
Python / FastAPI
Logica degli agenti AI, automazioni, integrazioni API esterne. Moderno e veloce.
Calendario ed eventi
Google Calendar API
Sincronizzazione bidirezionale con Supabase. Il team usa Google Calendar normalmente.
Deploy
Railway / Render
Deploy del backend Python. Piani gratuiti adeguati per la fase iniziale. Zero configurazione server.
